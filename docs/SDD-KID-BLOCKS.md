# SDD — 角色優先的機構積木環境（給小朋友的機構學）

> 聚焦設計文件。只涵蓋「角色優先積木環境」這條線，**不是**整個軟體的 SDD。
> 求解器、機構登錄、匯出、既有 wizard/進階模式一律不動或保留。

## 1. 目標與定位

對象：國中小／maker 入門的小朋友。
目的：用「機構學」的語言，引導他們**快速做出會動的機構原型**、發想。
精神：線上型的樂高——抓一塊零件、拖到場上、它自己卡住、按播放就會動。

設計北極星：**從「幾何優先」翻轉成「角色優先」。**
現況是：學生畫抽象的點/線，求解器在背後*推斷*角色（哪個是地、哪個有馬達）。
新方向是：機構學的**角色**才是主角，幾何是擺放零件後的結果。

## 2. 核心洞察：角色其實程式裡早就有

機構學的基本角色，程式裡已存在於「點的 type」與 bar 的 `isInput`，
經 `js/core/topology.js` `compileTopology` 編譯成求解步驟。本專案**只是把它重新呈現**，
不是重寫求解。

| 機構學概念（小朋友的話） | 既有程式表示 | 編譯成（solver step） |
|---|---|---|
| 🟤 機架／釘在地上的（固定桿） | 點 `type: 'fixed'` | `ground` |
| 🔴 動力－旋轉（圓）：馬達轉一圈 | 點 `type: 'motor'` / bar `isInput` | `input_crank` |
| 🟠 動力－直線（直）：推桿推出去 | 點 `type: 'linear'` / piston | `input_linear` |
| 🔵 從動－跟著動的（連桿／耦合件） | 點 `floating`（被解出） | `dyad` / `rigid_triangle` |

對照來源：`compileTopology` 的 ground 判定（fixed/motor/linear → ground）、
crank/linear 判定（motor 或 isInput → input_crank/input_linear）、
其餘 floating 點以 dyad 推斷。第一根桿的 P1 目前已**自動接地**
（`js/ui/wizard.js` `addLinkFromCanvas`）。

> 結論：**solver 一行都不用改。** 這是一層 UI + 詞彙 + 教學流程的「重新呈現」。

## 3. 零件目錄（把「角色」變成可抓的積木）

每個零件**長相即說明角色**；學生「抓哪塊」就等於回答了兩個關鍵決定
（① 釘住嗎？ ② 有沒有動力？），不必再問、不必藏一個下拉。

| 積木 | 角色 | 視覺符號 | 放下時自動做的事（對應既有資料） |
|---|---|---|---|
| 📌 地錨 | 機架/固定 | 斜線陰影（工程固定符號） | 在該點建立 `fixed` 點 → `ground` |
| 🔴↻ 馬達曲柄 | 旋轉動力 | 圓 + ↻ 旋轉箭頭 | 中心點 `motor`（自動釘地）+ 一根桿 `isInput` → `input_crank` |
| 🔵 連桿 | 從動 | 素桿 | 一般 bar，端點 `floating`，靠 dyad 解 |
| 🔺 三角板 | 從動/耦合 | 三角形 | `triangle` → `rigid_triangle` |
| ➡️ 推桿 | 直線動力 | → 直線箭頭 | `linear` / piston → `input_linear`（**本期暫不做**） |

## 4. 互動原則（貼近現代軟體習慣）

1. **零件盤 + 拖放 + 吸附**，取代「先選工具模式、再畫抽象點線」。
   （既有 canvas 的吸附/ghost line/拖曳合併可重用。）
2. **角色用顏色固定編碼**：機架=深色錨、旋轉動力=紅+↻、直線動力=橘+→、從動=藍。
   一眼讀出「每塊在做什麼」。
3. **「按下就會動」是核心循環**：湊齊「機架＋一個動力＋閉環」就讓一顆大 ▶ 發亮，
   按下去就跑（重用既有 theta/動畫）。
4. **失敗講機構學的人話**：動不了時說「還沒有馬達」「還沒釘在地上」「這兩根太短，搆不到」，
   而非 FAIL code（對接既有 `health-report` / diagnostics，只翻語氣）。
5. **慶祝軌跡（trace）這個 wow**：選一個點看它畫出什麼線（近似直線/弧/步態），
   這是機構學「一個輸入→豐富輸出」的靈魂。（重用既有 tracePoint。）

## 5. 第一刀範圍（MVP，先做、可驗證）

只放兩塊最關鍵的角色積木 + 播放，把「機架／旋轉動力／即時播放」一次串起來：

- 📌 **地錨** 與 🔴 **馬達曲柄** 兩塊積木的零件盤。
- 拖出馬達曲柄 → 自動把中心釘地、生成一根可轉的 `isInput` 曲柄。
- 一顆 ▶ 播放，按下去曲柄轉動。
- 底層**完全沿用** `fixed` / `motor` 點型與既有 solver、動畫。

**本期不做**：推桿(直線動力)、三角板、挑戰卡、失敗訊息改寫、軌跡慶祝動畫。
（驗證方向對了再逐步補。）

## 6. 漸進路線（之後）

1. 補 🔵 連桿、🔺 三角板（湊出第一個四連桿）。
2. 「按下就會動」的播放體驗 + 動不了時的人話提示。
3. 軌跡 wow：選點看畫出的線。
4. ➡️ 推桿（直線動力）。
5. 挑戰卡（用玩的方式帶 DOF：「再加一根會怎樣？」）。

## 7. 不碰 / 不破壞（紅線）

- **不改求解器** `js/multilink/solver.js`、`js/core/topology.js` 的數學。
- 既有 **wizard 進階模式、canvas 工具、匯出** 全部保留；新環境是**附加路徑**。
- 維持**零相依、純前端、ES6 module 靜態載入**，無 build step。
- 在 `feat/role-first-blocks` 分支開發；確認成形再合併 `main`（避免動到線上站）。

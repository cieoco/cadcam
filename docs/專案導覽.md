# 專案導覽（入門用，給人看的）

> 這份的目的：用最白話的方式，讓你在 5 分鐘內重新抓到「這專案是什麼、怎麼蓋的、做到哪、能往哪走」。
> 細節技術文件在最後一節有地圖；這份只負責「看懂全貌」。

---

## 1. 這是什麼？（用途）

一個**在瀏覽器裡設計「連桿機構」的工具**。連桿機構就是用桿子、轉軸、滑軌兜出來、會動的機械結構
（像汽車雨刷、伸縮燈、夾爪那種）。

你可以用它做三件事：

1. **畫** —— 擺零件、接起來，組成一個機構。
2. **模擬** —— 按播放，看它真的動起來、會不會卡住。
3. **輸出** —— 匯出 DXF（拿去雷射切割/CNC）或 `mechanism.json`（交給姊妹工具 `arm` 做 3D 組裝）。

它有**兩種使用面**：

- **機構工作頁**：選一種現成機構（四連桿、曲柄滑塊、齒條齒輪、平行四邊形、Jansen 腿…），調參數看它動。
- **blocks 積木沙盒**（`blocks.html`）：像樂高一樣**自由擺零件**自己兜機構。這是目前主力發展的方向，
  齒輪也是先在這裡長出來的。

**設計定位**：教學 / maker 用，UI 是繁體中文，強調「不用安裝、打開瀏覽器就能玩」。

---

## 2. 怎麼跑起來？

沒有安裝、沒有 npm、沒有 build。只要用 HTTP 開：

```bash
python -m http.server 8000
# 然後瀏覽器開 http://localhost:8000/mechanism.html  （機構工作頁）
#            或 http://localhost:8000/blocks.html      （積木沙盒）
```

（一定要用 HTTP，不能直接點開檔案——因為用了瀏覽器的 ES6 module 載入，`file://` 不支援。）

線上版掛在 GitHub Pages，所以 push 到 `main` 就等於更新線上 demo。

## 3. 怎麼蓋的？（心智模型，記這幾條就夠）

### (a) 純前端、零相依

整個專案就是一堆 `.js` 直接在瀏覽器跑，沒有打包工具、沒有套件。改完存檔、重整網頁就生效。

### (b) 最重要的一條界線：算數學的 vs 畫畫面的

```
   js/core/  ──「算」──   純函式，做數學/求解，不碰畫面
      │
      │  中間一個「引擎門面」mechanism-engine.js 把兩邊接起來
      │
   js/ui/    ──「畫」──   只管 DOM、滑鼠點擊，不做數學
```

> 規則：**core 不准碰畫面，ui 不准放求解數學。** 改東西時守住這條線，bug 會少很多。

### (c) 機構是「登錄表」驅動的

`js/mechanism-config.js` 像一張總表，每一種機構是一筆設定，寫好「它有哪些參數、用哪個 solver/畫圖/零件模組」。
**加一種新機構 = 在表裡加一筆 + 寫對應的三個檔（solver / visualization / parts）。** 不用改核心。

### (d) blocks 積木沙盒怎麼運作

零件種類：**地錨、連桿、三點桿、滑軌、齒輪、馬達**。你把它們擺上去、接起來，
底層會即時編譯成一個「拓樸（topology）」，丟給 **multilink 的求解器**算出每個關節的位置。

那個求解器用的是「**constructive geometry**（逐步建構幾何）」：從已知的固定點出發，
一個一個推算出其他點該在哪——適合單一自由度的機構。**這套數學是紅線，盡量別動。**

## 4. 做到哪了？（進度）

### 已經穩的

- **多種現成機構**：四連桿、曲柄滑塊、齒條齒輪、平行四邊形、通用 multilink、Jansen 腿、2D bar drawer。
- **blocks 沙盒的大重構**：把原本一大包的程式切成乾淨的小模組（畫圖、狀態、面板、工具、輸入…），
  並把效能瓶頸（每幀重畫整棵畫面）修掉。這部分已收斂、已驗證。

### 最近在做的：把 blocks 變成「機構繪製＋模擬系統」，開始加「基礎機件」

核心想法：**未來要陸續加入很多基礎機件（齒輪、齒條、凸輪、彈簧…），每個機件「連求解器一起進來」，
而且加一個機件 ≈ 加一張表項，不用改核心。** 這就是「機件外掛」架構。

**第一個真機件＝齒輪，已經落地：**

- 齒輪以**嚙合對**為單位（一個驅動、一個從動，會反向、按齒比轉）。
- **安裝方式比照連桿**（內部代號「方案乙」）：放下時是**浮動**的，由你**把齒輪中心拖去地錨/接點**才固定，
  跟接一根桿子一樣的操作邏輯。
- **要馬達才會轉**：沒給動力的齒輪就是個靜止的輪子（跟桿件一致）；要轉就把馬達放到驅動輪中心。
- 拖動時會**自動維持咬合**；如果硬把兩軸釘在對不上咬合距離的地方，齒輪會顯示**紅色虛線環**提醒。

**順手做的架構鋪路**：開始把「畫零件」的分派改成一張登錄表 `PART_DRAW`
（齒輪、三點桿已進表）。意思是日後加新機件，畫法就是「在表裡加一筆」。

> 如果你之前覺得「搞不清楚在忙什麼」——大方向就是這句：**讓齒輪這種機件，能像加表項一樣加進來。**

---

## 5. 未來可以往哪走？（走向）

由近到遠：

1. **把更多機件加進來**：齒條（架構已留位）、凸輪、彈簧…。每加一個 ≈ 一張表項 + 它的畫法/求解。
2. **把「求解」也做成登錄表**：目前「畫」已經表化了；「編譯成拓樸」和「求解」還是寫死的型別判斷。
   這兩個碰到核心檔案，要小心、一刀一型別慢慢做、每刀都要在瀏覽器驗證。
3. **教學 / 分享導向**：簡化課堂操作、分享連結讓學生互相打開、範例與學習卡。
4. **長線願景**：用自然語言描述 → AI 生成可用的機構（LLM 出設計、求解器當裁判，反覆迭代）。

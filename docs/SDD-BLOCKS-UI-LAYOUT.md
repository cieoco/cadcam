# SDD — blocks 實作界面規劃

> 對象：`blocks.html` / `js/blocks/` 的使用者介面。  
> 目標：範例與 3D 呈現收斂後，開始把 blocks 做成可長期使用的「機構實作界面」；同時支援桌機、平板、手機。

---

## 1. 北極星

blocks 的第一畫面應該是「可以直接做機構」的工作台，不是展示頁。

使用者進來後只需要理解四件事：

1. **放零件**：地錨、桿件、滑軌、齒輪、動力來源。
2. **接起來**：拖點吸附、對孔、對齒、對滑軌。
3. **調零件**：長度、齒數、角色、行程、疊層。
4. **看結果**：播放、軌跡、警告、2D/3D。

UI 的責任是把這四件事分清楚，讓按鈕不再平鋪成一串。

---

## 2. 現況問題

目前 `blocks.html` 已具備可用雛形：

- 上方：復原、存檔、開啟、分享、範例、回機構工具。
- 左側/手機底部：零件盤。
- 畫布浮層：長度、接點、伺服、齒輪、滑塊、活塞等編輯面板。
- 底部浮列：播放、角度、置中、孔、鎖、3D、清空。

問題也開始明顯：

- **功能分類不清**：檔案、建構、播放、視圖、診斷混在不同角落。
- **手機空間緊**：底部零件盤、播放列、浮動面板容易互相擋住。
- **面板太浮**：選到不同物件時，面板位置與內容分散，未來零件更多會失控。
- **桌機和平板沒有分工**：桌機可承受左右欄；平板需要抽屜/底部 sheet；手機需要極簡底部導航。
- **按鈕缺少優先級**：高頻操作和低頻操作同樣可見，之後加凸輪/皮帶輪/齒條會更擠。

---

## 3. 按鈕分類

所有按鈕先分成五群。實作時每顆按鈕都必須歸群，不能隨手塞進工具列。

| 群組 | 目的 | 代表按鈕 | 顯示原則 |
|---|---|---|---|
| 專案 | 檔案與範例 | 復原、存檔、開啟、分享、範例、回首頁 | 桌機在上方；手機收進「專案」選單 |
| 建構 | 放新零件 | 地錨、桿件、滑軌、齒輪、動力來源 | 永遠容易找到；手機用底部抽屜 |
| 編輯 | 編選取物 | 長度、齒數、行程、角色、疊層、刪除 | 只有選取後出現；手機用底部 sheet |
| 模擬/視圖 | 看動作與切視角 | 播放、角度、置中、孔、鎖、2D/3D、軌跡 | 播放永遠可見，其餘可收納 |
| 診斷/教學 | 告訴使用者為什麼 | DOF、死點、嚙合警告、任務卡 | 不搶操作；以狀態列/提示卡出現 |

危險操作規則：

- `清空`、`刪除` 不和主要建構按鈕同群。
- 手機上危險操作要在 sheet 內，避免誤觸。
- `刪除選取` 跟著選取面板走；`清空全部` 放專案/更多選單。

## 4. 版面模式

### 4.1 桌機：三區工作台

適用：寬度 `>= 1100px`。

```text
┌──────────────────────────────────────────────┐
│ App Bar: 專案 / 範例 / 分享 / 狀態             │
├──────────┬───────────────────────┬───────────┤
│ 建構盤   │ 2D/3D 工作畫布          │ 編輯檢查器 │
│ 左側欄   │ 底部浮動播放列          │ 右側欄     │
└──────────┴───────────────────────┴───────────┘
```

建議尺寸：

- 左側建構盤：`176-220px`，分段顯示「基礎」「傳動」「動力」。
- 右側檢查器：`300-360px`，顯示目前選取物的所有可編輯屬性。
- 畫布：吃剩餘空間，播放列浮在畫布底部中央。
- App Bar 高度：`48-56px`，只放專案級操作與整體狀態。

桌機上的按鈕呈現：

- 建構盤用「圖示 + 名稱 + 一行副標」。
- 右側檢查器用表單，不用散落浮窗。
- 底部播放列只保留播放、角度、置中、孔/鎖、3D。

### 4.2 平板：兩區 + 可收合面板

適用：`700px <= 寬度 < 1100px`。

```text
┌──────────────────────────┐
│ Compact App Bar           │
├────────┬─────────────────┤
│ 建構欄 │ 工作畫布          │
└────────┴──────┬──────────┘
                │ bottom inspector sheet
```

設計原則：

- 建構盤可在左側窄欄與底部抽屜之間切換。
- 編輯檢查器用底部 sheet，高度約 `36-52vh`，可收合。
- App Bar 不顯示所有專案按鈕；低頻操作進 overflow。
- 觸控目標至少 `44px`，常用建構按鈕至少 `52px`。

### 4.3 手機：畫布優先 + 底部工作列

適用：`< 700px` 或 coarse pointer。

```text
┌────────────────────┐
│ Top Bar: 返回/範例/更多 │
├────────────────────┤
│                    │
│      工作畫布        │
│                    │
├────────────────────┤
│ Play Strip          │
├────────────────────┤
│ Tabs: 建構 / 編輯 / 視圖 / 專案 │
└────────────────────┘
```

手機規則：

- 第一目標是保留畫布高度，避免零件盤長期吃掉太多空間。
- 底部固定四個 tab：`建構`、`編輯`、`視圖`、`專案`。
- 點 tab 後開 bottom sheet；未開 sheet 時只留薄底欄。
- `播放` 可以獨立成一條小浮列，永遠可用。
- 選取物件後自動切到 `編輯` sheet；sheet 可下滑關閉。
- 3D 是「檢視模式」，開啟後暫停 2D 編輯，避免手勢衝突。
- 沒有 hover 依賴；所有說明都要能以 title 以外的方式被看見。

手機安全區：

- 所有底部 UI 都要加 `env(safe-area-inset-bottom)`。
- sheet 展開時不可遮住播放列與系統手勢區。
- 橫向手機高度不足時，建構/編輯 sheet 只顯示兩行快捷項，其餘進「更多」。

## 5. 建構盤分類

未來零件會增加，所以零件盤不能只是一排卡片。建議分為三段：

### 基礎

- 地錨 / 固定孔
- 桿件
  - 連桿：兩個孔，固定長度
  - 三點桿：三個孔，剛性板
- 滑軌

### 傳動

- 齒輪對
- 齒條齒輪
- 皮帶輪
- 凸輪從動件

### 動力

- TT 馬達
- MG995 伺服
- 線性致動器

操作策略：

- 桌機：三段直接在左欄顯示，可用小標籤分段。
- 平板：顯示分類 tab，同一時間只顯示一段。
- 手機：底部 `建構` sheet 先顯示分類，再進入該分類的零件網格。
- 大類可再展開子類；例如先點 `桿件`，再選 `連桿` 或 `三點桿`，避免手機底部列塞太多相近按鈕。

---

## 6. 編輯檢查器分類

選取後右側欄 / bottom sheet 以物件類型切換內容，但內部順序固定：

1. **身份**：物件名稱、類型、顏色或圖示。
2. **幾何**：長度、齒數、模數、行程、座標。
3. **角色**：自由/固定/馬達/軌跡點。
4. **裝配**：疊層、馬達-機架-曲柄、固定端、翻面。
5. **操作**：複製、轉換、分離、刪除。

高頻調整用 stepper；連續數值未來再加 slider。  
手機上同一時間只展開一段，避免長面板壓住畫布。

---

## 7. 模擬與視圖列

播放列應該穩定，不跟零件盤混在一起。

永遠可見：

- 播放/暫停
- 目前角度

桌機/平板可見：

- 置中
- 孔
- 鎖
- 3D

手機收納到 `視圖`：

- 置中
- 顯示孔位
- 鎖定 8mm
- 2D/3D
- 清除軌跡 / 顯示軌跡

`孔` 與 `鎖` 是模式開關，應保留狀態顏色；`3D` 是檢視模式，開啟後應有明確返回 2D 的入口。

---

## 8. 斷點與尺寸規則

建議斷點：

| 名稱 | 條件 | 版面 |
|---|---|---|
| desktop | `min-width: 1100px` | 左建構欄 + 中畫布 + 右檢查器 |
| tablet | `700px-1099px` | 窄建構欄或底部抽屜 + bottom inspector |
| phone | `< 700px` 或 coarse pointer | 底部 tab + sheet |
| tiny phone | `< 380px` 或高度 `< 620px` | 文字縮短、只留圖示與必要值 |

尺寸規則：

- 觸控按鈕最小 `44x44px`。
- 主要建構按鈕手機建議 `64x56px`。
- 桌機小工具按鈕可 `32-36px`，但文字按鈕不可低於 `36px` 高。
- 所有 floating/sheet 面板最大高度要受 viewport 限制。
- 畫布可操作區不可被永久 UI 遮住超過 30% 高度。

## 9. 實作分期

### UI-P0：資訊架構先落文件

- 完成本文件。
- 在 `docs/專案導覽.md` 更新 roadmap。
- 盤點現有按鈕，全部歸到五大群組。

### UI-P1：HTML 結構重排但不改功能

- 新增語意容器：`appbar`、`workspace`、`part-palette`、`canvas-area`、`inspector`、`transport`。
- 保留既有 id 與 `window.blocks.*` 入口，避免一次改太多。
- 桌機先完成三區版面。

### UI-P2：檢查器集中化

- 把現有 `lenEditor`、`roleEditor`、`servoEditor`、`gearEditor`、`strokeEditor` 收斂到同一個 inspector 區。
- 桌機右側欄，平板/手機 bottom sheet。
- 先搬位置，不改每個控制項的行為。

目前實作狀態：

- `part-palette` 已先改成「基礎 / 傳動 / 動力」分類。
- `桿件` 已改成二級選單，再選 `連桿` 或 `三點桿`。
- 編輯面板已加上共用 `inspector-panel` class。
- 彈出選單已加上共用 `tool-menu` class，保留既有 `power-menu` 相容樣式。

### UI-P3：手機 bottom tabs / sheets

- 建立 `建構 / 編輯 / 視圖 / 專案` 四個 tab。
- 零件盤改成可收合 sheet。
- 選取物件時自動打開 `編輯` sheet。

目前實作狀態：

- 手機寬度已新增底部 `建構 / 編輯 / 視圖 / 專案` tabs。
- `建構` 會顯示既有零件盤；`編輯 / 視圖 / 專案` 先以輕量 sheet 承接常用按鈕。
- 選取連桿、三點桿、滑軌、齒輪、接點或放置動力後，手機會自動切到 `編輯` tab。
- 手機選取物件時，既有 inspector 會以 bottom sheet 形式貼在底部 tab 上方。
- 接點 inspector 已先整理成「角色」與「座標微調」兩段，避免手機上按鈕擠成單排。
- 目前尚未把 inspector 內容拆成更精緻的手機專用分頁，留到下一步處理。

### UI-P4：按鈕分類與 overflow

- 專案按鈕收進 App Bar / 專案 sheet。
- 低頻視圖按鈕進 `視圖`。
- 危險操作移到確認區，避免手機誤觸。

目前實作狀態：

- 手機版已將 `清空全部` 從播放控制列移到 `專案` sheet。
- 手機版 `清空全部` 會先確認，避免誤觸。
- 手機播放列只保留播放與角度；`置中 / 孔位 / 鎖孔距 / 3D` 收進 `視圖` sheet。

### UI-P5：診斷與教學入口

- 在畫布上方或 App Bar 顯示簡短狀態：可動 / 太鬆 / 卡住 / 缺馬達。
- 建立任務卡入口，但不做成 landing page。

## 10. 驗收

每個 UI phase 都要跑同一組黃金流程：

1. 載入每個 blocks 範例，播放 2D。
2. 切 3D 預覽，確認可旋轉/縮放，再回 2D。
3. 新建：連桿 + 地錨 + 馬達，播放。
4. 手機尺寸：新增連桿、拖曳接點、開/關 bottom sheet。
5. 平板尺寸：左欄/底部 sheet 不遮住主要操作。
6. 桌機尺寸：右側 inspector 不擋畫布。
7. 存檔、開啟、分享、復原都維持可用。

建議測試 viewport：

- `1366x768` 桌機
- `1024x768` 平板橫向
- `820x1180` 平板直向
- `390x844` 手機
- `360x640` 小手機

---

## 11. 非目標

- 不在 UI-P1/P2 同時新增新機件。
- 不改求解器。
- 不改資料格式，除非是已規劃的 `assembly` 顯示欄位。
- 不把 blocks 做成教學 landing page；第一屏仍是工作台。

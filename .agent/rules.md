# AI Coding Rules for this Project

## 1. Scope Preservation (最重要)
- **禁止「順便」修改**：嚴禁修改與當前任務無直接關聯的程式碼區塊。
- **維持現狀**：除非使用者要求，否則禁止進行重構（Refactoring）、排版優化或變數重新命名。
- **精確更動**：使用 `replace_file_content` 時，應盡量縮小 `TargetContent` 的範圍，僅更動必要的邏輯。

## 2. 修改前確認
- **提問優先**：如果修復 Bug 需要更動較大範圍的邏輯，必須先在對話中解釋計畫，得到確認後再執行。
- **影響評估**：在進行可能影響其他模組的更動時，須主動提醒使用者可能的副作用。

## 3. 程式碼風格
- **尊重原創**：嚴格遵循現有的縮排、空格、命名慣例與註解風格。
- **註解保留**：不要為了簡潔而刪除原有的中文或英文註解。

## 4. 測試與驗證
- **執行驗證**：每次修改後，應主動使用瀏覽器工具或測試腳本檢查是否影響到原本正常的功能。
